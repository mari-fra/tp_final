\begin{abstract}
Este trabajo presenta un análisis comparativo de diferentes modelos de Machine Learning aplicados al problema de clasificación propuesto por Mercado Libre en la competencia de Kaggle (Meli Challenge 2019). El objetivo principal es evaluar y comparar el rendimiento de diversos modelos contra la solución ganadora, considerando las limitaciones técnicas actuales para replicar la implementación original debido a dependencias obsoletas (TensorFlow 1.14, CUDA 10.0, Python 3.6/3.7) y problemas de compatibilidad con hardware y software modernos.
\end{abstract}

\section{Marco Teórico}

\subsection{Machine Learning para Clasificación de Texto}

% Agrega aquí la teoría relevante sobre clasificación de texto
La clasificación de texto es una tarea fundamental en el procesamiento de lenguaje natural (NLP) que consiste en asignar etiquetas o categorías a documentos de texto. En el contexto del Meli Challenge 2019, el objetivo es clasificar productos de e-commerce en categorías específicas basándose en sus descripciones.

\subsection{Arquitecturas de Redes Neuronales}

% Habla sobre LSTM, GRU, transformers, etc.
Las arquitecturas de redes neuronales profundas han demostrado ser efectivas para tareas de clasificación de texto. Entre las más utilizadas se encuentran:

\begin{itemize}
    \item \textbf{LSTM (Long Short-Term Memory):} Redes neuronales recurrentes capaces de capturar dependencias de largo plazo en secuencias de texto
    \item \textbf{GRU (Gated Recurrent Unit):} Variante más eficiente computacionalmente que LSTM
    \item \textbf{Transformers:} Arquitecturas basadas en mecanismos de atención que han revolucionado el NLP moderno
    \item \textbf{Embeddings:} Representaciones vectoriales densas de palabras o tokens que capturan relaciones semánticas
\end{itemize}

\section{Problemas Técnicos Encontrados}

\subsection{Limitaciones de la Solución Original}

La solución ganadora del 2019 presenta problemas de reproducibilidad debido a:

\begin{enumerate}
    \item \textbf{Dependencias obsoletas:}
    \begin{itemize}
        \item Python 3.6 / 3.7 (EOL - End of Life)
        \item TensorFlow 1.14
        \item CUDA 10.0 + cuDNN 7.6
    \end{itemize}
    
    \item \textbf{Incompatibilidades técnicas:}
    \begin{itemize}
        \item TensorFlow 1.14 no es compatible con GPUs modernas (RTX 30xx)
        \item CuDNNLSTM y CuDNNGRU requieren versiones específicas de CUDA/cuDNN que ya no están disponibles para drivers modernos
        \item Error: ``No OpKernel was registered to support Op 'CudnnRNN''
        \item Estas capas no tienen fallback a CPU, por lo que el entrenamiento falla inmediatamente sin GPU funcional
    \end{itemize}
    
    \item \textbf{Problemas con bibliotecas:}
    \begin{itemize}
        \item Incompatibilidad gensim + smart\_open
        \item SyntaxError con Python 3.6 por uso de \texttt{from \_\_future\_\_ import annotations}
        \item El paquete smart\_open utiliza anotaciones de tipo de Python 3.10+, no soportadas en Python 3.6
    \end{itemize}
    
    \item \textbf{Recursos computacionales:}
    \begin{itemize}
        \item Embeddings de gran tamaño (\textasciitilde6 GB)
        \item FastText requiere mucha RAM durante el entrenamiento
        \item Entrenamiento extremadamente lento en CPU
        \item Costo computacional elevado incluso con las tecnologías modernas
    \end{itemize}
\end{enumerate}

\subsection{Workarounds Implementados}

Para poder evaluar los modelos, se implementaron las siguientes soluciones alternativas:

\begin{itemize}
    \item Reemplazo de CuDNNLSTM/CuDNNGRU por LSTM/GRU estándar (permitiendo ejecución en CPU pero perdiendo la aceleración GPU específica de la solución original)
    \item Migración a TensorFlow 2.x o alternativas modernas
    \item Adaptación del código para ejecución en CPU
    \item Uso de versiones modernas de Python (3.8+) y bibliotecas compatibles
\end{itemize}

\textbf{Nota importante:} Estos workarounds permiten la ejecución funcional pero alteran la arquitectura original, haciendo imposible una replicación exacta de la solución ganadora del 2019 en entornos modernos.
